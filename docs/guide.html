mini-fibe — Interactive Guide

A self-hostable, single-tenant subset of the Fibe Platform as one Go binary + SQLite. Create Playspecs and Playgrounds and launch them on one startup-configured local Docker-host Marquee — speaking enough of the Fibe HTTP API that the real Fibe SDK and a curated slice of the Playwright API suite work unmodified.

Go single binary SQLite persistence Docker socket → Compose + Traefik Playguard reconcile loop SDK-compatible HTTP e2e adapters external

What fibe-distilled is (and isn't)

Fibe is a general-purpose, multi-tenant SaaS for launching app environments. fibe-distilled extracts the smallest reliable core of that — the create → launch → run loop on one Docker host — and deliberately drops everything multi-tenant, AI, and UI.

IS

  • A server-only JSON API compatible with a subset of the Fibe SDK / CLI / MCP.
  • Single owner, one static admin token.
  • Real local deploys: docker compose up, Traefik routing, real image builds.
  • Real runtime observation: container exit codes, health, logs read back through Docker.
  • An in-process Playguard loop (expiration, source sync, drift repair) — no external queue.

IS NOT

  • No browser UI, sessions, or Rails views.
  • No multi-tenancy, teams, players, billing, or API-key CRUD.
  • No agents / Genies / chats / autonomous coding sidecars.
  • No Bazaar / templates / template-versions / import flows.
  • No GitHub App / OAuth, secret vault, DNS automation, or Gitea.
Current status: the scoped runtime audit items are remediated on the real local Docker path. Health, builds, and lifecycle cleanup actions are driven by real container state and persisted evidence. Public DNS / real Let's Encrypt issuance remains outside the default harness, and scoped operational limitations are tracked separately — details in Status & known issues.

Quick start

The admin token, root domain, ACME email, Docker socket, and /opt/fibe mount are mandatory; the server refuses to boot without them.

Run from source

# required
export FIBE_API_KEY=dev-token
export FIBE_ROOT_DOMAIN=apps.example.com
export FIBE_ACME_EMAIL=ops@example.com
# optional
export FIBE_BUILD_PLATFORM=linux/amd64
export GITHUB_TOKEN=ghp_xxx
export GITHUB_WEBHOOK_SECRET=webhook-secret
export FIBE_PLAYGUARD_INTERVAL_SECONDS=30
export DOCKERHUB_USERNAME=... DOCKERHUB_TOKEN=...

go run ./cmd/fibe-distilled

Build & verify

# production binary
go build -tags sqlite_omit_load_extension -o ./bin/fibe-distilled ./cmd/fibe-distilled

# tests + alloy gate
go test ./...
./bin/check

Smoke test

curl -s http://localhost:2402/up.json                  # {"status":"ok"} (no auth)
curl -s -H 'Authorization: Bearer dev-token' \
     http://localhost:2402/api/me                      # fixed owner identity + SDK admin-scope hint
Point the Fibe SDK at fibe-distilled with FIBE_DOMAIN=http://localhost:2402 and FIBE_API_KEY=$FIBE_API_KEY. The same SDK calls for read-only Marquee discovery, Props, Playspecs, Playgrounds, launches, logs, and status work against the subset.

API layer

Fibe-shaped responses: {data,meta} list envelopes, {error:{code,message,details}}, X-Request-Id, name-or-ID lookup, async 202 + status_url polling.

Worker

Renders compose (+Traefik +x-fibe.gg vars), syncs git source, runs real local builds, deploys, then observes real container state.

Playguard

Ticker (default 30s): expiration reaper, source sync, runtime drift detect + repair.

Lifecycle & Playguard

Playground status is driven by real deploy steps and continuously reconciled against the host.

stateDiagram-v2
  [*] --> pending
  pending --> in_progress: deploy starts
  in_progress --> running: compose up OK
  in_progress --> error: build/deploy fail
  running --> stopping: stop
  stopping --> stopped
  stopped --> running: start
  running --> running: hard_restart (down + redeploy)
  running --> has_changes: dirty source on expiry/sync
  error --> running: drift repair / rollout
  running --> [*]: expire (destroy + delete)

Deploy steps (creation_steps)

compose_render → source_sync → builds → compose_deploy → runtime_observe → finalize. Each step is timestamped and surfaced in /status; a failed step preserves the playground in error state with classified diagnostics.

Playguard each tick

Per playground: enforce expiration (skip if source dirty), sync sources, re-inspect runtime, detect image/artifact drift and repair.
Latency knob: Playguard runs every FIBE_PLAYGUARD_INTERVAL_SECONDS seconds (default 30s; e2e harness uses 1s). A POST /status can force a live runtime refresh when clients need immediate state.

Unsupported Fibe surfaces

fibe-distilled is a normal-Playground runtime subset. Full Fibe job-mode/Tricks calls fail fast before business handlers run.

sequenceDiagram
  participant C as Client
  participant API as API
  C->>API: POST /api/playgrounds {job_mode:true, marquee_id}
  API-->>C: 501 NOT_IMPLEMENTED
  C->>API: POST /api/tricks
  API-->>C: 501 NOT_IMPLEMENTED
Rejected job-mode inputs: job_mode, target_type=trick, fibe.gg/job_watch, services[].job_watch, only_services, except_services, services._run, result_status, and /api/tricks.

Source builds

For services with a build: + fibe.gg/repo_url, the source is cloned on the Marquee and built remotely — with verified BuildRecord reuse.

flowchart LR
  A["build: service + fibe.gg/repo_url"] --> B["git clone / pull source on Marquee"]
  B --> C{"prior BuildRecord image exists for commit + identity digest?"}
  C -- yes --> D["reuse image (Reused=true)"]
  C -- no --> E["docker build with commit SHA + identity labels"]
  E --> F["BuildRecord: success/failed + real logs + image ref"]
  D --> F
  F --> G["image ref injected into rendered compose"]
Build identity = digest of {dockerfile, target, build_args}. Stale builds (stuck > 45m) are recovered to success only if a host image with the matching commit + identity digest actually exists; otherwise they are marked failed. DockerHub creds (if set) are written to a per-playground docker-config to avoid anonymous pull rate limits.

Usage flows

Copy-paste recipes. Set TOK='Authorization: Bearer dev-token' and BASE=http://localhost:2402 first.

# Configure the single Marquee before starting fibe-distilled
export FIBE_API_KEY=dev-token
export FIBE_ROOT_DOMAIN=apps.example.com
export FIBE_ACME_EMAIL=ops@example.com
export FIBE_BUILD_PLATFORM=linux/amd64  # optional

go run ./cmd/fibe-distilled

# Discover it through the read-only Fibe-compatible API
curl -s $BASE/api/marquees -H "$TOK"
curl -s $BASE/api/marquees/default -H "$TOK"
# Create a Playspec from compose; fibe-distilled validates compose inside create/update
curl -s -X POST $BASE/api/playspecs -H "$TOK" -H 'Content-Type: application/json' -d '{
  "playspec":{"name":"web-spec",
  "base_compose_yaml":"services:\n  web:\n    image: nginx\n    labels:\n      fibe.gg/subdomain: web\n"}}'

curl -s $BASE/api/playspecs/web-spec/services -H "$TOK"   # extracted services
# Create + deploy a Playground (deploy runs inside the request), then poll to running
curl -s -X POST $BASE/api/playgrounds -H "$TOK" -H 'Content-Type: application/json' -d '{
  "playground":{"name":"web-pg","playspec_id":<PLAYSPEC_ID>,
  "env_overrides":{"FOO":"bar"}}}'                                  # 201

curl -s $BASE/api/playgrounds/web-pg/status -H "$TOK"               # status, service_urls, build_statuses
curl -s -X POST $BASE/api/playgrounds/web-pg/status -H "$TOK"       # force live re-inspect
# Full Fibe job-mode/Tricks calls fail fast in fibe-distilled.
curl -s -X POST $BASE/api/playgrounds -H "$TOK" -H 'Content-Type: application/json' -d '{
  "playground":{"name":"job-pg","playspec_id":<PLAYSPEC_ID>,"marquee_id":<MARQUEE_ID>,"job_mode":true}}'

# => 501 NOT_IMPLEMENTED. job-mode/Tricks are Full Fibe only in this extraction.

# Launch from caller-supplied compose with optional GitHub source metadata; repo must be writable when supplied
curl -s -X POST $BASE/api/launches -H "$TOK" -H 'Content-Type: application/json' -d '{
  "name":"repo-backed-web",
  "repository_url":"https://github.com/owner/repo",
  "compose_yaml":"services:\n  web:\n    image: nginx\n    labels:\n      fibe.gg/subdomain: web\n",
  "create_playground":true}'
# => {playspec_id, playground_id, props_created, source}. Non-writable repo => 422 REPOSITORY_REQUIRES_FORK
# Drive fibe-distilled with the real Fibe Go SDK
export FIBE_DOMAIN=http://localhost:2402     # SDK base URL
export FIBE_API_KEY=dev-token

# Go: client := fibe.NewClient()  // reads FIBE_DOMAIN / FIBE_API_KEY
# client.Playspecs / client.Playgrounds / read-only client.Marquees / Launches / Logs work against the subset
# docker-e2e lives in ../fibe-distilled-e2e: fibe-distilled + e2e-proxy + dind Marquee + sdk + playwright + runtime-smoke + results
cd ../fibe-distilled-e2e
export FIBEDISTILLED_APP_PATH=../fibe-distilled
GH=$(gh auth token)
GITHUB_PAT="$GH" GITHUB_TOKEN="$GH" \
DOCKERHUB_USERNAME=... DOCKERHUB_TOKEN=... \
./bin/docker-e2e
# fibe-distilled image is built normally; e2e-proxy owns SDK/Playwright bootstrap compatibility.
# Narrow: GO_TEST_RUN=... GO_TEST_RUNS=... PLAYWRIGHT_TEST_TARGETS=... PLAYWRIGHT_TEST_GREP=...

e2e & module isolation

Docker-e2e bootstrap adapters live in fibe-distilled-e2e. fibe-distilled keeps only production routes. The fake runtime executor is a Go test helper and is not imported by the production binary.

flowchart TD
  SRC["source tree"] --> PROD["go build -tags sqlite_omit_load_extension -o $prod_bin ./cmd/fibe-distilled"]
  SRC --> TEST["go test ./..."]
  PROD --> P1["production routes only"]
  P1 --> P2["/e2e_backdoor/* absent"]
  P1 --> P3["no internal/e2e import · no fixture marker shim"]
  E2E["fibe-distilled-e2e"] --> E1["e2e-proxy"]
  E1 --> E2X["/e2e_backdoor/bootstrap + /operation"]
  E1 --> E3["fixture_marker rewritten before forwarding"]
Do not expose the e2e proxy outside local test networks. It exists only so unchanged SDK/Playwright bootstrap helpers can target fibe-distilled during docker-e2e.

Remediated audit items

OK #1 — Lifecycle cleanup is exclusive-root scoped
the configured Marquee and /opt/fibe/playgrounds root are fibe-distilled managed
Destructive operations validate the compose project/path before touching Compose, containers, networks, volumes, or /opt/fibe. Playguard removes safe remote project directories absent from SQLite because the configured Marquee is no longer shared with other owners.
OK #2 — Job-mode/Tricks are removed from scope
compatgate returns NOT_IMPLEMENTED for job-mode inputs
Fibe-distilled no longer runs or simulates Tricks. job_mode, fibe.gg/job_watch, run selectors, result_status, and /api/tricks fail fast.
OK #6 — Image/artifact repair has cooldown metadata
playguard_repair_reason · playguard_repair_lock_until · needs_recreation
Runtime drift repair now persists a cooldown and exposes pending recreation in status payloads, preventing every-tick redeploy loops while keeping the operator-visible state honest.

Current accepted tradeoffs

MED Container hardening is trusted-operator scope
fibe-distilled validates supported Fibe labels and Compose shape, not container isolation
privileged, network_mode:host, pid:host, /var/run/docker.sock, and sensitive host bind mounts are accepted for Playgrounds authored by the single trusted operator. Fibe-distilled must not be exposed as a multi-tenant untrusted-Compose service.
MED No full optimistic-lock/CAS layer on playground rows
internal/storage · internal/worker/refresh.go · internal/playground/http.go
SQLite uses one open connection, WAL, and busy_timeout; lifecycle handlers and Playguard re-read rows at important boundaries. Broader simultaneous JSON-field conflicts remain best-effort under the single-process runtime.
LOW Public DNS and ACME issuance are live-environment checks
the default docker-e2e harness proves local HTTPS/SNI routing, not public HTTP-01 reachability
Fibe-distilled always emits HTTPS/Let's Encrypt Traefik configuration for the configured Marquee. Operators still need real DNS pointing at a publicly reachable host before browser-trusted certificates can be issued.
Net verdict: the minimal scoped server is correct and shippable for the stated single-tenant, server-only scope. The external fibe-distilled-e2e default proves local HTTPS/SNI routing, owned /opt/fibe artifacts, and exclusive-root lifecycle cleanup. Real public DNS and Let's Encrypt issuance remain the final live-domain smoke outside the default private Docker harness.
